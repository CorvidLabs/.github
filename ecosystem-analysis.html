New Micro Package Ideas

Small, focused packages that DON'T duplicate 0xLeif's existing work:

swift-once Micro
Execute code exactly once. Thread-safe lazy init, one-time setup, feature flags that fire once.
Different from Later: Simpler API, focused only on one-time execution pattern
Claude Prompt
Create swift-once - execute code exactly once, following CorvidLabs conventions. Core type: Once - Actor-based for thread safety - Lazy evaluation - only runs when first accessed - Caches result for subsequent calls - Supports async and throwing Additional: - OnceFlag: Bool that can only become true once - @Once property wrapper Example: ```swift let setup = Once { try await Database.connect() } let db = try await setup() // Connects first time let db2 = try await setup() // Returns cached let migrated = OnceFlag() if migrated.trigger() { migrate() } // Only once ``` ~50 lines. Zero dependencies.
swift-throttle Micro
Rate limiting and debouncing. Control execution frequency for APIs, UI events.
Not in 0xLeif repos: Focused throttle/debounce/rate-limit patterns
Claude Prompt
Create swift-throttle - rate limiting utilities, following CorvidLabs conventions. Core types: 1. Throttle - Execute at most once per interval 2. Debounce - Execute after quiet period 3. RateLimiter - Token bucket algorithm Example: ```swift let log = Throttle(interval: 1.0) { print($0) } log("a") // Prints log("b") // Dropped (within interval) let search = Debounce(delay: 0.3) { query in await performSearch(query) } search("help") // Only executes after 300ms quiet let api = RateLimiter(rate: 10, per: .second) if await api.acquire() { makeAPICall() } ``` ~150 lines. Zero dependencies. Sendable.
swift-tap Micro
Side-effect chaining. Inspect values without breaking fluent APIs. Debug logging made easy.
Different from Closure: One-liner side effects, not full function composition
Claude Prompt
Create swift-tap - side-effect chaining, following CorvidLabs conventions. Extensions on common types: - tap(_:) - Execute side effect, return self - debug(_:) - Print with label, return self - measure(_:) - Time execution, return self Example: ```swift let user = fetchUser() .tap { print("Fetched: \($0.id)") } .tap { analytics.track("user_fetched") } let result = complexCalculation() .debug("result") // Prints: [DEBUG] result: 42 let data = loadData() .measure("loading") // Prints timing ``` ~30 lines. Zero dependencies.
swift-either Micro
Either type for dual returns. When you need A or B, not Optional or Result.
Not in 0xLeif repos: Classic functional Either<Left, Right>
Claude Prompt
Create swift-either - Either type, following CorvidLabs conventions. Core: Either - .left(Left) or .right(Right) - Map/flatMap for both sides - Fold to extract value - Swap sides - Convert to/from Result Example: ```swift func fetch() -> Either { cache.exists ? .left(cached) : .right(fresh) } let user: User = result.fold( left: { $0.toUser() }, right: { $0.toUser() } ) ``` ~80 lines. Sendable, Equatable, Hashable.
swift-with Micro
Configure objects inline. Kotlin's 'with' and 'apply' for Swift.
Not in 0xLeif repos: Simple inline configuration helper
Claude Prompt
Create swift-with - inline configuration, following CorvidLabs conventions. Functions: - with(_:_:) - Configure and return (reference types) - mutating(_:_:) - Configure value types Example: ```swift let label = with(UILabel()) { $0.text = "Hello" $0.font = .systemFont(ofSize: 16) $0.textColor = .label } let rect = CGRect.zero.mutating { $0.size.width = 100 $0.size.height = 50 } ``` ~20 lines. The simplest useful package.
swift-id Micro
Type-safe identifiers. Prevent mixing UserID with PostID. Phantom types.
Not in 0xLeif repos: Compile-time ID type safety
Claude Prompt
Create swift-id - type-safe identifiers, following CorvidLabs conventions. Core protocol: TypedID - Wraps raw value (UUID, String, Int) - Codable as raw value - Hashable, Equatable, Sendable Example: ```swift struct UserID: TypedID { let rawValue: UUID } struct PostID: TypedID { let rawValue: UUID } func fetchUser(id: UserID) -> User func fetchPost(id: PostID) -> Post fetchUser(id: userId) // OK fetchUser(id: postId) // Compile error! ``` ~50 lines. Compiler-enforced safety.
swift-lens Micro
Functional getters/setters for nested data. Immutable updates without boilerplate.
Not in 0xLeif repos: Functional optics pattern
Claude Prompt
Create swift-lens - functional optics, following CorvidLabs conventions. Core: Lens - Get and set nested values - Compose for deep nesting - KeyPath-based construction - Immutable updates Example: ```swift struct User { var profile: Profile } struct Profile { var name: String } let nameLens = Lens(\User.profile.name) let name = nameLens.get(user) let updated = nameLens.set(user, "Jane") // Composition let deep = userLens.compose(profileLens) ``` ~100 lines. Value semantics.
swift-pool Micro
Resource pooling. Reuse expensive objects (connections, buffers).
Not in 0xLeif repos: Connection/resource pool pattern
Claude Prompt
Create swift-pool - resource pooling, following CorvidLabs conventions. Core: Pool - Fixed or dynamic size - Acquire/release pattern - Auto cleanup on idle - Health checks Example: ```swift let pool = Pool( maxSize: 10, create: { try await Database.connect() }, destroy: { try await $0.close() } ) try await pool.withResource { conn in try await conn.query("SELECT ...") } // Auto released ``` Actor-based. ~120 lines.
